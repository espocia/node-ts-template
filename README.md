# node-ts-template

A Node.js REST API template built with Express 5, TypeScript, and PostgreSQL.

## Prerequisites

- [Node.js](https://nodejs.org/) (LTS version)
- [Docker](https://www.docker.com/) (for PostgreSQL)

## Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment

Create a `.env` file in the project root:

```env
PGUSER=root
PGPASSWORD=root
PGDATABASE=postgres
PGHOST=localhost
PGPORT=5432
DB_CONNECTION=postgres://root:root@localhost:5432/postgres
```

### 3. Create the Docker volume

The Postgres container uses an external Docker volume. Create it once:

```bash
docker volume create learning-node
```

### 4. Start the database

This starts the Postgres container and generates `postgres-language-server.jsonc` for IDE SQL support:

```bash
npm run db:init
```

### 5. Initialize the schema

Source your `.env` then use `psql` directly:

```bash
export $(grep -v '^#' .env | xargs)
psql -f src/db/initialize_table.sql
```

## Running

**Development** (hot-reload via nodemon):

```bash
npm run dev
```

**Production**:

```bash
npm run build
npm start
```

The server listens on `http://localhost:3000`.

## Database

| Script | Description |
|---|---|
| `npm run db:init` | Start Postgres container + generate LSP config |
| `npm run db:nuke` | Stop container and remove LSP config |

## API Endpoints

### `POST /user`

Creates a new user (in-memory, not persisted).

**Request:**
```json
{ "name": "John Doe", "email": "john@example.com" }
```

**Response:**
```json
{ "id": "abc123", "name": "John Doe", "email": "john@example.com", "createdAt": "..." }
```

### `GET /users`

Returns all users from the database.

### `GET /users/:id`

Returns a single user by ID.

## Tech Stack

- **Runtime:** Node.js (LTS)
- **Framework:** Express 5
- **Language:** TypeScript
- **Database:** PostgreSQL 18 (Docker)
- **DB Client:** `pg` with connection pooling
- **Dev Tools:** nodemon, ts-node, ESLint, Prettier

## PostgreSQL LSP Setup

The `postgres-language-server.jsonc` in the project root is auto-generated by `npm run db:init` and contains the connection config the LSP uses for SQL intelligence (autocomplete, type checking, query validation).

### VSCode

1. Install the [Postgres Language Server](https://marketplace.visualstudio.com/items?itemName=Supabase.postgres-lsp) extension from the VSCode marketplace.
2. Run `npm run db:init` to generate `postgres-language-server.jsonc`.
3. Open the project folder — the extension auto-detects the config file.

### Neovim (Mason)

Since you've already installed `postgres-language-server` via Mason, you just need to wire it up in your LSP config.

**With `nvim-lspconfig`:**

```lua
require('lspconfig').postgres_lsp.setup({})
```

The LSP will walk up the directory tree and find `postgres-language-server.jsonc` automatically.

**With `mason-lspconfig` (auto-setup):**

```lua
require('mason-lspconfig').setup({
  ensure_installed = { 'postgres_lsp' },
})

require('mason-lspconfig').setup_handlers({
  ['postgres_lsp'] = function()
    require('lspconfig').postgres_lsp.setup({})
  end,
})
```

**Verify the LSP is active:**

Open a `.sql` file or a TypeScript file with an inline SQL string, then run:

```
:LspInfo
```

You should see `postgres_lsp` listed as an active client. If the server fails to connect, make sure the Postgres container is running (`docker compose ps`) and `postgres-language-server.jsonc` exists in the project root.

**Filetype associations:**

`postgres_lsp` attaches to `sql` filetypes by default. If you want it active in TypeScript files (for template literal SQL), check the extension's docs for tagged template support.
